# Snapshot 003: ODPM Ontology and Knowledge Structure

Date: 2026-06-30

Snapshot #003 turns the ODPM manuscript into a clearer knowledge structure.

The core model remains simple:

```text
PRD
(Intent + Requirements + Assumptions + Constraints)

    v

Ontology
(Shared Model of the World)

    v

Design
Architecture
Implementation
Testing
```

## What changed

- `BOOK.md` now carries the manifesto-style ODPM manuscript.
- `questions.md` separates foundational questions from open research questions.
- `glossary.md` defines the core terms used across the repository.
- Principle files remain concise concept cards.

## Current structure

- `README.md` explains the repository.
- `BOOK.md` is the current manuscript.
- `questions.md` preserves questions that drive understanding.
- `glossary.md` keeps terminology aligned.
- `principles/` holds concise concept cards.
- `snapshots/` captures the evolution of understanding.
- `examples/` holds applied domain examples.

## Terminology

The current core terms are:

- Ontology
- Concept
- Relationship
- Rule
- State
- Shared Understanding
- Decision Readiness
- Chase Understanding
- Snapshot

## Working standard

The repository should stay intentionally simple.

New files should exist only when they preserve understanding better than expanding an existing file.
