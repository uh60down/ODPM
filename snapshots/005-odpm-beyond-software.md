# Snapshot 005: ODPM Beyond Software

Date: 2026-06-30

ODPM began as a framework for software product teams.

The three-layer model, Chase Understanding, and Decision Readiness are native to that context.

But the underlying logic is not exclusive to software.

## The Claim

Any shared work that fails due to misalignment — not poor execution — can benefit from ODPM.

Project management. Change management. Organizational design. Strategy execution.

In all of these, the root failure is the same: people are executing from different models of the world.

## The Project Ontology

A project — any project — can be modeled using the same four building blocks.

| Building Block | Project Example |
|---|---|
| Concepts | Stakeholder, Deliverable, Decision, Risk, Constraint, Milestone |
| Relationships | Stakeholder approves Deliverable; Risk blocks Decision; Milestone gates next phase |
| Rules | No phase gate without Decision Readiness; no delivery without sign-off; escalation requires documented options |
| States | Proposed / In Progress / Blocked / Approved / Done / Cancelled |

Building this model before building the project plan forces the team to agree on what exists before agreeing on what to do.

## What This Changes

Traditional project management produces a plan, then executes it.

ODPM-informed project management builds a shared model first, then produces a plan that executes that model.

The plan becomes downstream of understanding, not a substitute for it.

## The Change Management Extension

Change management has a specific version of this:

- Current-state ontology: the world as it exists before the change
- Target-state ontology: the world as it will exist after the change
- The change plan: the path between them

See: principles/change-ontology.md and principles/resistance-is-an-ontology-gap.md

## What Stays the Same

Chase Understanding still applies. Questions reveal reality.

Decision Readiness still applies. The goal is not complete understanding. It is enough understanding to act responsibly.

Snapshot Thinking still applies. Every plan and model is a current best expression of reality.

The tools change. The principles do not.
