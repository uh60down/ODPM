# Sprint Ontology Review

A lightweight review to run alongside Scrum ceremonies. ODPM sits around Scrum, not inside it — this template improves the quality of what enters the ceremonies, without changing them.

Copy this file per sprint. Keep it short.

> Sprint: _()_
> Date: _(YYYY-MM-DD)_

---

## Before the sprint — Definition of Ready

A story is Decision Ready when it references the shared ontology.

- [ ] Each story references established Concepts, Relationships, Rules, States, and Actions — not undefined ones.
- [ ] Stories that introduce an undefined concept are labelled `open-question` and have a Chase Understanding session scheduled.

## Sprint Planning

- [ ] Not only "do we have capacity?" but "is our ontology aligned enough to act responsibly on this scope?"
- [ ] Open questions blocking the sprint goal are in the [Open Question Log](open-question-log.md) with owners.

## Sprint Review

- [ ] The increment is reviewed against the ontology, not only against acceptance criteria.
- [ ] Where the implementation diverged from the shared model, it is named. Decide: change the implementation, or update the ontology.

## Sprint Retrospective

- [ ] Ask: "What did we learn about the world this sprint?"
- [ ] If the answer changes the shared model, update the ontology before the next sprint begins (a natural snapshot moment).

## Bugs as Ontology Signals

For recurring bugs, ask:

- [ ] Was this a coding error? → fix the code.
- [ ] Was the world model wrong? → fix the ontology, then the code.
