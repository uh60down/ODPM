# Decision Readiness Checklist

Decision Readiness is observable, not a feeling. Use this checklist for a specific decision, not the project in general.

> Decision: _(the next responsible action being considered)_
> Date: _(YYYY-MM-DD)_

---

A team is Decision Ready when it can do all four of these out loud:

- [ ] **Point to the model.** We can name the concepts, relationships, rules, states, and actions this decision touches.
- [ ] **Show what is uncertain.** The open questions that remain are written down, not carried in someone's head.
- [ ] **Show who owns each uncertainty.** Every remaining open question has a name against it in the [Open Question Log](open-question-log.md).
- [ ] **Explain why the unknowns no longer block.** We can say why the remaining questions do not prevent this specific decision.

---

## If any box is unchecked

The team is not yet Decision Ready. That is not a failure — it is a signal for where to Chase Understanding next.

- Unchecked box 1 → the ontology isn't extracted yet. Run the [Ontology Canvas](ontology-canvas.md).
- Unchecked box 2 → uncertainty is implicit. Write the open questions down.
- Unchecked box 3 → questions have no home. Assign owners.
- Unchecked box 4 → the block is real. Keep chasing, or narrow the decision until the remaining unknowns no longer block it.

---

> Decision Readiness does not require certainty. It requires enough clarity that the remaining unknowns no longer prevent the next responsible move.
